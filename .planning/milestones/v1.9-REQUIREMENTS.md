# Requirements: VST3 WebView UI Designer

**Defined:** 2026-01-29
**Core Value:** Visually design a plugin UI and export code that works in JUCE WebView2 without manual fixups.

## v1.9 Requirements

Requirements for Layers & Help System milestone. Maps to Phases 41-43.

### Bug Fixes

- [x] **BUG-01**: Single-window folder export writes directly to selected folder (no subfolder)
- [x] **BUG-02**: Container editor multi-select drag moves all selected elements together

### Layers Panel

- [x] **LAYER-01**: User can see all elements in current window as a layer list
- [x] **LAYER-02**: Layer list shows element name and type icon for each element
- [x] **LAYER-03**: User can toggle element visibility via eye icon (hidden elements don't render)
- [x] **LAYER-04**: User can toggle element lock via lock icon (locked elements can't be moved/resized)
- [x] **LAYER-05**: User can drag layers to reorder (changes z-order on canvas)
- [x] **LAYER-06**: Clicking a layer selects the element on canvas
- [x] **LAYER-07**: Selecting element on canvas highlights it in layers panel
- [x] **LAYER-08**: User can double-click layer name to rename element inline
- [x] **LAYER-09**: Layers panel appears as tab in LeftPanel (alongside Elements, Assets)

### Help System

- [x] **HELP-01**: Each Properties Panel section has help (?) button in header
- [x] **HELP-02**: Clicking help button opens new browser window with documentation
- [x] **HELP-03**: Help content explains the section's properties with examples
- [x] **HELP-04**: Help window has dark theme matching app style
- [x] **HELP-05**: Help content exists for each element type (Knob, Slider, Button, etc.)
- [x] **HELP-06**: User can press F1 to open help for currently selected element
- [x] **HELP-07**: Help window shows step-by-step instructions where applicable

## Future Requirements

Deferred to v2.0+:

### Layers Enhancements
- **LAYER-10**: Group layers without changing element hierarchy
- **LAYER-11**: Multi-select in layers with Ctrl/Shift click
- **LAYER-12**: Show nested hierarchy for containers with children

### Help Enhancements
- **HELP-08**: Searchable help index page
- **HELP-09**: Cross-linked help topics

## Out of Scope

| Feature | Reason |
|---------|--------|
| Layer groups (folders) | Complexity - elements already have containers |
| Context menu in layers | v2.0 enhancement |
| External help website | Offline use required, bundled content better |
| Video tutorials in help | Scope creep, text+images sufficient |
