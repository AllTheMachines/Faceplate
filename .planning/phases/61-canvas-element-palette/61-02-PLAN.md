**2. Section: Using the Palette** (anchored as `## Using the Palette`)

Brief overview of palette mechanics:
- The palette is in the **Elements** tab of the left panel (selected by default)
- Elements are organized into collapsible categories -- click a category header to expand or collapse it
- The first three categories are expanded by default
- To add an element, drag it from the palette onto the canvas (see [Canvas](canvas.md) for details)

Screenshot placeholder: `![Element palette showing categories and search bar](../images/palette-overview.png)`

**3. Section: Search & Filter** (anchored as `## Search & Filter`)

Describe the search functionality:
- Type in the search bar at the top of the palette to filter elements by name
- The search updates as you type with a brief delay for responsiveness
- A count of matching elements appears below the search bar
- All matching categories expand automatically during search
- Clear the search by clicking the X button or deleting the text

**5. Section: Pro Elements** (anchored as `## Pro Elements`)

Overview: Some elements require a Faceplate Pro license. These are marked with a **(Pro)** badge in the palette and in the tables above.

Explain gating behavior:
- Without a Pro license, Pro elements are hidden from the palette by default
- Pro users can toggle the "Hide Pro elements" switch at the top of the palette to show or hide Pro elements
- If you load a project that contains Pro elements without a Pro license, those elements will display a Pro badge overlay on the canvas
- Pro elements in an unlicensed project have read-only properties -- you can view settings but not change them
- Export to JUCE is blocked when the project contains Pro elements and no valid Pro license is active
- For licensing details, see the License settings in the application

Note on counts: There are 50 Pro elements across the Meters, Visualizations, Curves, Navigation & Selection, Images & Decorative, Linear Controls, Buttons, and Specialized Audio categories.
